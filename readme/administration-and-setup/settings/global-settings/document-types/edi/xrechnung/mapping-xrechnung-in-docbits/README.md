# Zuordnung von XRechnung in DocBits

In DocBits werden XRechnung-Rechnungen mithilfe einer vordefinierten Konfiguration bestimmten Feldern zugeordnet, um sicherzustellen, dass die Daten nahtlos in verschiedene Formate exportiert werden können, einschließlich der Integration mit anderen Systemen wie **Infor**. Die Exportkonfiguration nutzt Vorlagen und Regeln, um sicherzustellen, dass jedes Element der XRechnung erfasst und angemessen zugeordnet wird.

### Hauptkomponenten

1.  **Dokumenttypen**: XRechnung-Dokumente werden bestimmten **Dokumenttypen** in DocBits zugeordnet. Jeder Dokumenttyp (z. B. Rechnung, Gutschrift, Lastschrift) hat seine eigene Struktur und Felder.

2.  **Feldzuordnung**: Felder in der XRechnung werden mithilfe einer **Exportkonfigurationsdatei** entsprechenden Feldern in DocBits zugeordnet. Diese Datei definiert, wie jedes XRechnung-Feld behandelt wird und wohin es exportiert werden soll.

3.  **Exportregeln**: Bestimmte Regeln werden definiert, um spezielle Fälle zu behandeln, in denen Werte abweichen können, einschließlich Toleranzprüfungen, Genehmigungsanforderungen oder Gebühren auf Zeilenebene. Diese Regeln stellen sicher, dass XRechnung-Daten basierend auf spezifischer Geschäftslogik korrekt verarbeitet und exportiert werden.
