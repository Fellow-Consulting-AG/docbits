# Mappage de XRechnung dans DocBits

Dans DocBits, les factures XRechnung sont mappées à des champs spécifiques à l'aide d'une configuration prédéfinie qui garantit que les données peuvent être exportées de manière transparente vers divers formats, y compris l'intégration avec d'autres systèmes comme **Infor**. La configuration d'exportation s'appuie sur des modèles et des règles pour garantir que chaque élément de la XRechnung est capturé et mappé de manière appropriée.

### Composants clés

1.  **Types de documents** : Les documents XRechnung sont mappés à des **Types de documents** spécifiques dans DocBits. Chaque type de document (par exemple, facture, avoir, note de débit) a sa propre structure et ses propres champs.

2.  **Mappage des champs** : Les champs de la XRechnung sont mappés aux champs correspondants dans DocBits à l'aide d'un **fichier de configuration d'exportation**. Ce fichier définit comment chaque champ XRechnung est géré et vers où il doit être exporté.

3.  **Règles d'exportation** : Certaines règles sont définies pour gérer des cas spécifiques où les valeurs peuvent différer, notamment les contrôles de tolérance, les exigences d'approbation ou les frais au niveau de la ligne. Ces règles garantissent que les données XRechnung sont traitées et exportées correctement, sur la base d'une logique métier spécifique.
