# Mapowanie XRechnung w DocBits

W DocBits faktury XRechnung są mapowane na określone pola przy użyciu wstępnie zdefiniowanej konfiguracji, która zapewnia, że dane mogą być płynnie eksportowane do różnych formatów, w tym integracji z innymi systemami, takimi jak **Infor**. Konfiguracja eksportu wykorzystuje szablony i reguły, aby zapewnić, że każdy element XRechnung zostanie przechwycony i odpowiednio zmapowany.

### Kluczowe komponenty

1.  **Typy dokumentów**: Dokumenty XRechnung są mapowane na określone **typy dokumentów** w DocBits. Każdy typ dokumentu (np. faktura, nota kredytowa, nota debetowa) ma własną strukturę i pola.

2.  **Mapowanie pól**: Pola w XRechnung są mapowane na odpowiadające im pola w DocBits przy użyciu **pliku konfiguracji eksportu**. Ten plik definiuje, jak każde pole XRechnung jest obsługiwane i gdzie powinno zostać wyeksportowane.

3.  **Reguły eksportu**: Zdefiniowano pewne reguły obsługi konkretnych przypadków, w których wartości mogą się różnić, w tym kontrole tolerancji, wymagania dotyczące zatwierdzania lub opłaty na poziomie linii. Reguły te zapewniają, że dane XRechnung są przetwarzane i eksportowane poprawnie, w oparciu o specyficzną logikę biznesową.
