# Configuración de Propiedades de Campo

## La configuración de las propiedades de campo es fundamental para asegurar que los datos capturados en un documento cumplan con los requisitos específicos y los procesos de procesamiento.

### Aquí están los pasos para configurar las propiedades de campo para satisfacer necesidades específicas de procesamiento de datos:

#### Seleccionar un tipo de campo:

* Comience seleccionando el tipo de campo adecuado para sus datos.

<figure><img src="../../../../../.gitbook/assets/docbits_field_properties_type.png" alt="Docbits Field Properties Type" width="375"><figcaption></figcaption></figure>

* Esto depende del tipo de información que contendrá el campo.

<figure><img src="../../../../../.gitbook/assets/docbits_field_properties_panel.png" alt="Docbits Field Properties Panel" width="139"><figcaption><p>Propiedades</p></figcaption></figure>

* Los tipos de campo posibles incluyen texto, número, fecha, menú desplegable, casilla de verificación, etc.

<figure><img src="../../../../../.gitbook/assets/image (146).png" alt="" width="241"><figcaption><p>Elementos</p></figcaption></figure>

#### Establecer validación y formato:

* Establezca reglas de validación para asegurar que los datos introducidos cumplan con los criterios esperados.
* Esto puede incluir la comprobación de ciertos patrones de cadenas, límites numéricos, formatos de fecha u otras condiciones.

<figure><img src="../../../../../.gitbook/assets/image (147).png" alt="" width="236"><figcaption></figcaption></figure>

#### Definir valores predeterminados:

* Si ciertos campos suelen tener un valor predeterminado, puede establecerlo como valor predeterminado.
* Esto facilita la entrada de datos porque los usuarios no tienen que introducir el mismo valor cada vez.

<figure><img src="../../../../../.gitbook/assets/image (148).png" alt="" width="224"><figcaption></figcaption></figure>

#### Establecer permisos de usuario:

* Determine qué grupos de usuarios deben tener acceso al campo y qué tipo de derechos de acceso tienen.

<figure><img src="../../../../../.gitbook/assets/image (150).png" alt=""><figcaption><p>Configuración: Grupos y Permisos</p></figcaption></figure>

* Esto puede incluir derechos de lectura, escritura o edición.

<figure><img src="../../../../../.gitbook/assets/image (151).png" alt=""><figcaption><p>Ver Permisos</p></figcaption></figure>

#### Vincular a otros datos:

* En algunos casos, los datos de un campo deben vincularse a los datos de otro campo o fuente de datos.
* Configure los vínculos o relaciones apropiados para asegurar una integración de datos consistente.

#### Aplicar reglas de visibilidad:

* Determine bajo qué condiciones un campo debe ser visible u oculto.
* Esto puede ser útil para adaptar dinámicamente la interfaz de usuario basándose en ciertos datos o acciones del usuario.

<figure><img src="../../../../../.gitbook/assets/image (149).png" alt=""><figcaption><p>Configuración de Campos</p></figcaption></figure>

#### Habilitar historización:

* Si es necesario, habilite la historización de campos para rastrear los cambios históricamente.
* Esto le permite rastrear los cambios en los datos y supervisar el historial de cambios de datos.

#### Añadir notas de documentación:

* Añada notas o descripciones para explicar a los usuarios cómo usar el campo o qué tipo de datos se esperan.

Siguiendo estos pasos y configurando las propiedades de campo adecuadas, puede asegurar que sus documentos cumplan con los requisitos específicos para el manejo de datos, el acceso de usuarios y la precisión de los datos.
