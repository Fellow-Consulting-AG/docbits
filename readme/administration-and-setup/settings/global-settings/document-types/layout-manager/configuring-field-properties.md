# Konfigurieren von Feldeigenschaften

## Die Konfiguration von Feldeigenschaften ist entscheidend, um sicherzustellen, dass die in einem Beleg erfassten Daten spezifische Anforderungen und Verarbeitungsprozesse erfüllen.

### Hier sind die Schritte zum Konfigurieren von Feldeigenschaften, um spezifische Anforderungen an die Datenverarbeitung zu erfüllen:

#### Wählen Sie einen Feldtyp aus:

* Beginnen Sie mit der Auswahl des richtigen Feldtyps für Ihre Daten.

<figure><img src="../../../../../.gitbook/assets/Bildschirmfoto 2024-05-24 um 11.06.36.png" alt="" width="375"><figcaption></figcaption></figure>

* Dies hängt davon ab, welche Art von Informationen das Feld enthalten wird.

<figure><img src="../../../../../.gitbook/assets/Bildschirmfoto 2024-05-24 um 11.06.56.png" alt="" width="139"><figcaption><p>Eigenschaften</p></figcaption></figure>

* Mögliche Feldtypen sind Text, Zahl, Datum, Dropdown-Menü, Kontrollkästchen usw.

<figure><img src="../../../../../.gitbook/assets/image (146).png" alt="" width="241"><figcaption><p>Elemente</p></figcaption></figure>

#### Validierung und Formatierung festlegen:

* Legen Sie Validierungsregeln fest, um sicherzustellen, dass die eingegebenen Daten den erwarteten Kriterien entsprechen.
* Dies kann die Prüfung auf bestimmte Zeichenfolgenmuster, numerische Grenzwerte, Datumsformate oder andere Bedingungen beinhalten.

<figure><img src="../../../../../.gitbook/assets/image (147).png" alt="" width="236"><figcaption></figcaption></figure>

#### Standardwerte definieren:

* Wenn bestimmte Felder normalerweise einen Standardwert haben, können Sie diesen als Standardwert festlegen.
* Dies erleichtert die Dateneingabe, da Benutzer nicht jedes Mal denselben Wert eingeben müssen.

<figure><img src="../../../../../.gitbook/assets/image (148).png" alt="" width="224"><figcaption></figcaption></figure>

#### Benutzerberechtigungen festlegen:

* Legen Sie fest, welche Benutzergruppen Zugriff auf das Feld haben sollen und welche Art von Zugriffsrechten sie haben.

<figure><img src="../../../../../.gitbook/assets/image (150).png" alt=""><figcaption><p>Einstellungen: Gruppen und Berechtigungen</p></figcaption></figure>

* Dies kann Lese-, Schreib- oder Bearbeitungsrechte beinhalten.

<figure><img src="../../../../../.gitbook/assets/image (151).png" alt=""><figcaption><p>Berechtigungen anzeigen</p></figcaption></figure>

#### Link zu anderen Daten:

* In einigen Fällen müssen Daten aus einem Feld mit Daten aus einem anderen Feld oder einer anderen Datenquelle verknüpft werden.
* Konfigurieren Sie entsprechende Links oder Beziehungen, um eine konsistente Datenintegration sicherzustellen.

#### Sichtbarkeitsregeln anwenden:

* Legen Sie fest, unter welchen Bedingungen ein Feld sichtbar oder ausgeblendet sein soll.
* Dies kann nützlich sein, um die Benutzeroberfläche basierend auf bestimmten Daten oder Benutzeraktionen dynamisch anzupassen.

<figure><img src="../../../../../.gitbook/assets/image (149).png" alt=""><figcaption><p>Felder einstellen</p></figcaption></figure>

#### Historisierung aktivieren:

* Aktivieren Sie bei Bedarf die Historisierung von Feldern, um Änderungen historisch zu verfolgen.
* Dies ermöglicht es Ihnen, Änderungen an den Daten zu verfolgen und den Verlauf von Datenänderungen zu überwachen.

#### Dokumentationsnotizen hinzufügen:

* Fügen Sie Notizen oder Beschreibungen hinzu, um den Benutzern zu erklären, wie das Feld zu verwenden ist oder welche Art von Daten erwartet wird.

Durch Befolgen dieser Schritte und Konfigurieren der entsprechenden Feldeigenschaften können Sie sicherstellen, dass Ihre Belege spezifische Anforderungen an die Datenhandhabung, den Benutzerzugriff und die Datengenauigkeit erfüllen.
