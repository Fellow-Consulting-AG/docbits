# Configuration des propriétés de champ

## La configuration des propriétés de champ est essentielle pour garantir que les données saisies dans un document répondent aux exigences spécifiques et aux processus de traitement.

### Voici les étapes pour configurer les propriétés de champ afin de répondre aux besoins spécifiques de traitement des données :

#### Sélectionner un type de champ :

* Commencez par sélectionner le type de champ approprié pour vos données.

<figure><img src="../../../../../.gitbook/assets/docbits_field_properties_type.png" alt="Docbits Field Properties Type" width="375"><figcaption></figcaption></figure>

* Cela dépend du type d'informations que le champ contiendra.

<figure><img src="../../../../../.gitbook/assets/docbits_field_properties_panel.png" alt="Docbits Field Properties Panel" width="139"><figcaption><p>Propriétés</p></figcaption></figure>

* Les types de champs possibles incluent le texte, le nombre, la date, le menu déroulant, la case à cocher, etc.

<figure><img src="../../../../../.gitbook/assets/image (146).png" alt="" width="241"><figcaption><p>Éléments</p></figcaption></figure>

#### Définir la validation et le formatage :

* Définissez des règles de validation pour garantir que les données saisies répondent aux critères attendus.
* Cela peut inclure la vérification de certains modèles de chaînes, de limites numériques, de formats de date ou d'autres conditions.

<figure><img src="../../../../../.gitbook/assets/image (147).png" alt="" width="236"><figcaption></figcaption></figure>

#### Définir des valeurs par défaut :

* Si certains champs ont généralement une valeur par défaut, vous pouvez la définir comme valeur par défaut.
* Cela facilite la saisie des données car les utilisateurs n'ont pas à saisir la même valeur à chaque fois.

<figure><img src="../../../../../.gitbook/assets/image (148).png" alt="" width="224"><figcaption></figcaption></figure>

#### Définir les autorisations des utilisateurs :

* Déterminez quels groupes d'utilisateurs doivent avoir accès au champ et quel type de droits d'accès ils possèdent.

<figure><img src="../../../../../.gitbook/assets/image (150).png" alt=""><figcaption><p>Paramètres : Groupes et autorisations</p></figcaption></figure>

* Cela peut inclure des droits de lecture, d'écriture ou d'édition.

<figure><img src="../../../../../.gitbook/assets/image (151).png" alt=""><figcaption><p>Voir les autorisations</p></figcaption></figure>

#### Lier à d'autres données :

* Dans certains cas, les données d'un champ doivent être liées aux données d'un autre champ ou d'une autre source de données.
* Configurez les liens ou relations appropriés pour assurer une intégration cohérente des données.

#### Appliquer des règles de visibilité :

* Déterminez dans quelles conditions un champ doit être visible ou masqué.
* Cela peut être utile pour adapter dynamiquement l'interface utilisateur en fonction de certaines données ou actions de l'utilisateur.

<figure><img src="../../../../../.gitbook/assets/image (149).png" alt=""><figcaption><p>Paramétrage des champs</p></figcaption></figure>

#### Activer l'historisation :

* Si nécessaire, activez l'historisation des champs pour suivre les modifications historiquement.
* Cela vous permet de suivre les modifications apportées aux données et de surveiller l'historique des changements de données.

#### Ajouter des notes de documentation :

* Ajoutez des notes ou des descriptions pour expliquer aux utilisateurs comment utiliser le champ ou quel type de données est attendu.

En suivant ces étapes et en configurant les propriétés de champ appropriées, vous pouvez vous assurer que vos documents répondent aux exigences spécifiques en matière de manipulation des données, d'accès des utilisateurs et d'exactitude des données.
