# Configurando Propriedades de Campo

## Configurar as propriedades do campo é fundamental para garantir que os dados capturados em um documento atendam a requisitos e processos de processamento específicos.

### Aqui estão as etapas para configurar as propriedades do campo para atender às necessidades específicas de processamento de dados:

#### Selecionar um tipo de campo:

* Comece selecionando o tipo de campo correto para seus dados.

<figure><img src="../../../../../.gitbook/assets/docbits_field_properties_type.png" alt="Docbits Field Properties Type" width="375"><figcaption></figcaption></figure>

* Isso depende de que tipo de informação o campo conterá.

<figure><img src="../../../../../.gitbook/assets/docbits_field_properties_panel.png" alt="Docbits Field Properties Panel" width="139"><figcaption><p>Propriedades</p></figcaption></figure>

* Os tipos de campo possíveis incluem texto, número, data, menu suspenso, caixa de seleção, etc.

<figure><img src="../../../../../.gitbook/assets/image (146).png" alt="" width="241"><figcaption><p>Elementos</p></figcaption></figure>

#### Definir validação e formatação:

* Defina regras de validação para garantir que os dados inseridos atendam aos critérios esperados.
* Isso pode incluir a verificação de certos padrões de string, limites numéricos, formatos de data ou outras condições.

<figure><img src="../../../../../.gitbook/assets/image (147).png" alt="" width="236"><figcaption></figcaption></figure>

#### Definir valores padrão:

* Se certos campos normalmente têm um valor padrão, você pode defini-lo como o valor padrão.
* Isso facilita a entrada de dados porque os usuários não precisam digitar o mesmo valor todas as vezes.

<figure><img src="../../../../../.gitbook/assets/image (148).png" alt="" width="224"><figcaption></figcaption></figure>

#### Definir permissões de usuário:

* Determine quais grupos de usuários devem ter acesso ao campo e que tipo de direitos de acesso eles têm.

<figure><img src="../../../../../.gitbook/assets/image (150).png" alt=""><figcaption><p>Configurações: Grupos e Permissões</p></figcaption></figure>

* Isso pode incluir direitos de leitura, gravação ou edição.

<figure><img src="../../../../../.gitbook/assets/image (151).png" alt=""><figcaption><p>Ver Permissões</p></figcaption></figure>

#### Vincular a outros dados:

* Em alguns casos, os dados de um campo precisam ser vinculados aos dados de outro campo ou fonte de dados.
* Configure links ou relacionamentos apropriados para garantir uma integração de dados consistente.

#### Aplicar regras de visibilidade:

* Determine sob quais condições um campo deve estar visível ou oculto.
* Isso pode ser útil para adaptar dinamicamente a interface do usuário com base em determinados dados ou ações do usuário.

<figure><img src="../../../../../.gitbook/assets/image (149).png" alt=""><figcaption><p>Configuração de Campos</p></figcaption></figure>

#### Ativar historização:

* Se necessário, ative a historização de campos para rastrear alterações historicamente.
* Isso permite rastrear alterações nos dados e monitorar o histórico de alterações de dados.

#### Adicionar notas de documentação:

* Adicione notas ou descrições para explicar aos usuários como usar o campo ou que tipo de dados é esperado.

Ao seguir essas etapas e configurar as propriedades de campo apropriadas, você pode garantir que seus documentos atendam a requisitos específicos para manuseio de dados, acesso do usuário e precisão dos dados.
