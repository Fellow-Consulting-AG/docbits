# Modulo attivo

## **Scopo:**

Questa carta DocBits controlla se uno specifico modulo nel sistema è attivo o inattivo. Consente ai flussi di lavoro di procedere in base allo stato di attivazione di un modulo, garantendo che le azioni vengano eseguite solo se il modulo necessario è disponibile.

## **Funzionalità:**

* **Convalida dello stato del modulo:** Questa carta verifica lo stato di attivazione di uno specifico modulo e lo valuta rispetto a una condizione definita dall'utente.
* **Selezione del modulo:** Gli utenti specificano il nome del modulo da controllare, garantendo una convalida precisa.
* **Operatori:** Possono essere applicate le seguenti condizioni:
  * **È (Is):** Il flusso di lavoro continua se il modulo selezionato è attivo.
  * **Non è (Is Not):** Il flusso di lavoro continua se il modulo selezionato è inattivo.

## **Utilizzo:**

Questa carta è particolarmente utile per gli amministratori o i gestori di sistema che devono creare flussi di lavoro dipendenti dalla disponibilità o dalla funzionalità di moduli specifici. Aiuta a garantire che i flussi di lavoro vengano eseguiti solo quando tutti i moduli richiesti sono configurati in modo appropriato.

## **Esempio di scenario**

* Un utente configura la carta per controllare se il modulo **"Elaborazione documenti"** **è attivo.** Se il modulo è attivo, il flusso di lavoro procede, attivando attività automatizzate di elaborazione dei documenti. Se il modulo è inattivo, il flusso di lavoro si arresta, impedendo azioni non necessarie.

Utilizzando la carta "Controllo modulo attivo", le organizzazioni possono migliorare l'affidabilità del flusso di lavoro, evitare errori dovuti a moduli inattivi e garantire che i processi siano allineati alla configurazione del sistema.
